# QEMU Embedded Decompression Profiler

Desktop-speed, embedded-accurate iteration on decompressor performance. Builds a
bare-metal benchmark firmware for several Cortex-M cores, runs it under
`qemu-system-arm` (MPS2 machines) with a custom TCG plugin that counts every
executed guest instruction per PC, and reports symbolized per-function profiles.

The headline metric is **core insns/byte**: executed instructions attributed to
decompressor code, divided by decompressed bytes. Single vCPU, no interrupts, so
runs are bit-for-bit deterministic — any delta between two builds is real, with
no averaging or reruns needed.

QEMU is not cycle-accurate: it does not model caches, dual-issue, or branch
prediction, and multi-register `ldm`/`stm` count as one instruction. Use the
instruction counts to iterate, then confirm promising changes on real hardware
(`make stm32h7b0-device-benchmark`, `make rp2040-device-benchmark`).

## Requirements

- `brew install qemu` (11.x; provides `qemu-system-arm` and `qemu-plugin.h`)
- Arm GNU Toolchain (`arm-none-eabi-gcc`) on PATH
- `datasets/enwik8` (`make download-enwik8`) — only for the `extended` variant;
  the `classic` variant reuses the checked-in `devices/rp2040` data headers

## Usage

```bash
python3 tools/qemu-profiler/profile.py                    # m0plus, m4, m7; classic blob
python3 tools/qemu-profiler/profile.py --variants extended
python3 tools/qemu-profiler/profile.py --json build/base.json
python3 tools/qemu-profiler/profile.py --cflags "-DTAMP_FAST_WINDOW_COPY=1" --json build/exp.json
python3 tools/qemu-profiler/profile.py --compare build/base.json build/exp.json

# Per-instruction annotated disassembly of the most recent run:
python3 tools/qemu-profiler/profile.py --cores m7 --annotate tamp_decompressor_decompress_cb

# Replay a host fuzz corpus through the real ARM binaries on the emulated cores:
python3 tools/qemu-profiler/profile.py --replay fuzz/corpus_decompressor
```

`--replay` runs each corpus entry through `firmware/replay_main.c` (same
config/chunk-byte scheme as `fuzz/fuzz_decompressor.c`) on every selected core.
Host ASAN fuzzing proves the C paths; the replay catches ISA/ABI-specific
defects it cannot — target codegen at `-O3`, unsigned plain `char`, 32-bit
`size_t`. There is no sanitizer on bare metal, so the window/output buffers are
canary-fenced (checked after every entry) and faults trip the vector handlers.

Cores map to QEMU machines: `m0plus`/`m3` → mps2-an385 (armv6m code runs fine on
the emulated M3), `m4` → mps2-an386, `m7` → mps2-an500. The m4/m7 builds use
`TAMP_ARMV7EM=1` like the STM32H7B0 harness; m0plus is the portable build
(RP2040-like).

The workload matches `devices/BENCHMARKS.md`: one-shot full-buffer decompress of
the first 100 KB of enwik8, window=10. `classic` is the v1-format blob;
`extended` is generated by the tamp CLI at build time. The firmware memcmps the
output against the original bytes each run, so every profile is also a
correctness check.

## Layout

- `plugin/insncount.c` — TCG plugin; per-PC executed-instruction histogram
- `firmware/` — vector table, semihosting glue, benchmark `main`, linker script
- `profile.py` — build + run + symbolize + compare driver
- `build/` — firmware ELFs, raw counts, result JSONs (gitignored)
